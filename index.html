Conceitos Básicos

O que é Git?

Git é um sistema de controle de versão distribuído que permite rastrear alterações no código-fonte durante o desenvolvimento de software. Foi criado por Linus Torvalds em 2005 para o desenvolvimento do kernel Linux.

O que é GitHub?

GitHub é uma plataforma de hospedagem de código-fonte baseada em Git. Ele adiciona recursos próprios ao Git, incluindo interface web, controle de acesso, ferramentas de colaboração, e muito mais.

Principais vantagens:

  • Controle de histórico de alterações
  • Trabalho em equipe facilitado
  • Ramificação (branching) e fusão (merging) eficientes
  • Rastreamento de issues e solicitações de pull

Estratégia de Três Branches

Uma estratégia comum de branches envolve três ramificações principais. Esta abordagem permite um desenvolvimento organizado, com código estável na branch principal e isolamento de novas funcionalidades.

Main/Master

Branch principal que contém o código de produção. Deve estar sempre estável e pronto para implantação.

Como criar:

# A branch main é criada por padrão ao inicializar um repositório
git init
git add .
git commit -m "Initial commit"

# Ou ao clonar um repositório existente
git clone https://github.com/usuario/repositorio.git
Develop

Branch de desenvolvimento onde as funcionalidades são integradas antes de irem para produção.

Como criar:

# Criar a branch develop a partir da main
git checkout main
git pull origin main
git checkout -b develop

# Enviar para o repositório remoto
git push -u origin develop
Feature

Branches temporárias criadas a partir de develop para desenvolver novas funcionalidades específicas.

Como criar:

# Criar uma branch de funcionalidade
git checkout develop
git pull origin develop
git checkout -b feature/nome-da-funcionalidade

# Desenvolver a funcionalidade e fazer commits
git add .
git commit -m "Implementa nova funcionalidade"

# Enviar para o repositório remoto
git push -u origin feature/nome-da-funcionalidade

Fluxo entre branches:

  1. Novas funcionalidades são desenvolvidas em branches de feature
  2. Quando prontas, são mescladas na branch develop
  3. Após testes e estabilização, develop é mesclada em main para produção
  4. Correções de bugs críticos podem ser feitas através de hotfix branches

Fluxo de Trabalho no Git/GitHub

Um fluxo de trabalho comum no Git/GitHub usando a estratégia de três branches envolve os seguintes passos:

  1. Clonar o repositório: Copiar um repositório remoto para sua máquina local
  2. Sincronizar com a branch develop: Manter-se atualizado com as mudanças da equipe
  3. Criar uma branch de feature: Isolar suas alterações para uma nova funcionalidade
  4. Fazer commits regularmente: Salvar alterações no histórico da branch
  5. Fazer push: Enviar commits para o repositório remoto
  6. Abrir um Pull Request: Solicitar a integração das alterações na develop
  7. Revisão e merge: Revisar o código e integrar à branch develop
  8. Mesclar develop em main: Após testes, levar as alterações para produção
# Fluxo básico de trabalho com branches
git clone https://github.com/usuario/repositorio.git
cd repositorio

# Sincronizar com a develop
git checkout develop
git pull origin develop

# Criar branch de funcionalidade
git checkout -b feature/nova-funcionalidade

# Desenvolver e fazer commits
git add .
git commit -m "Implementa nova funcionalidade"

# Enviar para o repositório remoto
git push -u origin feature/nova-funcionalidade

# Após aprovação, mesclar com develop
git checkout develop
git merge feature/nova-funcionalidade
git push origin develop

Resolução de Conflitos

Conflitos ocorrem quando o Git não pode mesclar automaticamente alterações diferentes no mesmo arquivo. Eles devem ser resolvidos manualmente.

Como resolver conflitos:

  1. Identifique os arquivos com conflitos (git status)
  2. Abra cada arquivo e localize os marcadores de conflito
  3. Decida qual código manter (ou combine as alterações)
  4. Remova os marcadores de conflito
  5. Adicione os arquivos resolvidos (git add)
  6. Complete o merge (git commit)
# Durante um merge com conflitos

# Verificar arquivos com conflitos
git status

# Após resolver os conflitos manualmente
git add arquivo.txt
git commit -m "Resolvendo conflitos de merge"

Prevenindo conflitos:

  • Comunique-se com sua equipe sobre alterações em andamento
  • Faça commits pequenos e frequentes
  • Atualize sua branch regularmente com as alterações da branch principal
  • Use ferramentas de diff/merge para visualizar alterações
  • Resolva conflitos assim que surgirem, não os deixe acumular
