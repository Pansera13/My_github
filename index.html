Conceitos Básicos do Git e GitHub

Git é um sistema de controle de versão distribuído que permite acompanhar mudanças no código-fonte durante o desenvolvimento de software.

GitHub é uma plataforma de hospedagem de código-fonte que utiliza o Git para controle de versões, com recursos adicionais de colaboração.

Principais Conceitos:

  • Repositório: Local onde são armazenados todos os arquivos e histórico de um projeto
  • Commit: Snapshot das alterações feitas no repositório em um determinado momento
  • Branch: Ramificação do código que permite desenvolver funcionalidades isoladamente
  • Merge: Ação de unir o código de diferentes branches
  • Pull Request: Solicitação para incorporar alterações de uma branch em outra
# Inicializar um repositório Git git init # Clonar um repositório existente git clone <url-do-repositório> # Verificar o status dos arquivos git status

Fluxo de Trabalho no Git/GitHub

Um fluxo de trabalho comum no Git envolve as seguintes etapas:

Fazer alterações nos arquivos
Adicionar alterações (git add)
Fazer commit (git commit)
Enviar para remoto (git push)

Comandos do Fluxo Básico:

# Adicionar arquivos específicos ao staging git add arquivo1.txt arquivo2.js # Adicionar todos os arquivos modificados git add . # Fazer commit com mensagem git commit -m "Mensagem descritiva do commit" # Enviar commits para o repositório remoto git push origin main

Como Resolver Conflitos no Git

Conflitos ocorrem quando o Git não pode mesclar automaticamente as alterações de diferentes branches ou commits.

Etapas para Resolver Conflitos:

  1. Identificar os arquivos com conflitos (git status)
  2. Abrir cada arquivo e localizar as marcações de conflito
  3. Decidir qual código manter (ou combinar as alterações)
  4. Remover as marcações de conflito do Git
  5. Adicionar os arquivos resolvidos ao staging (git add)
  6. Completar o merge (git commit)

Exemplo de arquivo com conflito:

<<<<<<< HEAD console.log("Versão local"); ======= console.log("Versão remota"); >>>>>>> branch-remota

Como ficaria após a resolução:

console.log("Versão combinada");

Trabalhando com Branches

Branches permitem desenvolver funcionalidades, corrigir bugs ou experimentar ideias sem afetar a codebase principal.

Branch Main/Master

A branch principal que contém o código de produção. Deve sempre estar estável e pronta para deploy.

# Criar uma nova branch a partir da main git checkout -b nova-feature main # Atualizar a branch main com as últimas alterações git checkout main git pull origin main

Branch Develop

Branch de desenvolvimento onde as funcionalidades são integradas antes de irem para produção.

# Mudar para a branch develop git checkout develop # Integrar uma feature na develop git checkout develop git merge feature-nova

Branch Feature

Branches temporárias para desenvolver novas funcionalidades. São criadas a partir da develop e depois mescladas de volta.

# Criar uma branch de feature git checkout -b minha-feature develop # Trabalhar na feature e fazer commits git add . git commit -m "Desenvolvendo nova funcionalidade" # Finalizar a feature e mesclar à develop git checkout develop git merge --no-ff minha-feature git branch -d minha-feature
